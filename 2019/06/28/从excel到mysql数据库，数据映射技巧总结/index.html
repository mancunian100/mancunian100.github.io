从excel到mysql数据库，数据映射技巧总结

读取excel这种格式的数据遇到了很多问题和麻烦，总结的技巧如下

  • 技巧总结：

  1. 在处理空值和0时，可以一起处理，读出字符串，判断长度是否大于1（默认非0值的字符串长度都大于1）

  2. 在处理使用＊作为分割符号时，可以用split方法取出字符串数组，若数组长度为2就说明有分割，若为1则默认数量为1

  3. 面对合并单元格的excel数据，需要对这种情况进行特殊处理，根据excel的行号特点可以使用如下方法：
    在某一列可能出现合并单元格时，由于excel在行号处理时仍然会将这一行在数值上分隔开，所以可以先使用一个临时变量记录上一行的值，若下一行值为空，则仍使用临时变量中存储的值，直到下一次遇到非空的行

  4. 由于qualification这一列数据的特殊性，需要分三种情况讨论数据的分类，分别是数据属于cost，属于status，属于trade，分别在三张表中进行查询，获取对应id后再写入
